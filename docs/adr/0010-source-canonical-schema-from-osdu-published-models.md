# Source the canonical schema from OSDU-published models, not hand-authored names

**Context.** Slice #2's canonical tables were hand-authored to look "PDM-shaped" but matched no OSDU/PPDM name verbatim, and the reference's whole value rests on genuine OSDU grounding. Primary-source verification (2026-07) established: OSDU is an independent Open Group / OSDU Forum standard — **not** derived from PPDM, though it incorporates PPDM reference lists and leverages Energistics domain standards; the OSDU **Production DDMS ("OSDU PDM v1.0", Halliburton contribution, Apache-2.0)** is an explicitly PPDM-3.9-based **relational** model whose **full column-level Data Dictionary is openly published**; and the OSDU **WKS JSON schemas** (`osdu/data/data-definitions`) are **Apache-2.0** and vendorable.

**Decision.** Canonical table/column names are **sourced from the OSDU PDM v1.0 published Data Dictionary** (the openly-licensed relational model) — not invented, and not from the licence-gated PPDM dictionary — with each slice populating only the **subset of entities/fields its use case needs** and validating against that dictionary (the OSDU-conformance acceptance criteria on issues #4/#6/#8, and semantic-mapping conformance on #3). The **secondary OSDU JSON manifest** (ADR 0007) is generated to the **OSDU WKS schemas**. Both source specs are **vendored into `spec/osdu/`** (pinned, with Apache-2.0 attribution), and the canonical schema is defined **once** in a table-spec module that generation, gold, and the writer all reference. The generator's expected-rate forecast (ADR 0006) is aligned to OSDU PDM's **Decline Analysis** entities / the `Forecast` quantity-method rather than treated as bespoke. PPDM 3.9 is cited as lineage only.

**Why.** Makes OSDU conformance **verifiable by construction** against an authoritative, openly-licensed source; removes the invented-name divergence found in slice #2; preserves ADR 0007's canonical (PDM relational) / secondary (WKS JSON) split; and sidesteps PPDM licensing entirely.
