# Axis-A demonstrations emerge from the graded contest; one designated platform per round (round 1: Databricks); no independent reference instantiation

**Context.** The contest issues already have every contestant instantiating the semantic/agent layers on a real platform, so an independent maintainer-built reference instantiation (old issue #11) duplicated graded work and publicly exposed agent/LPG implementations for contest themes before the contest ran (answer-key leakage). Separately, letting contestants pick different stacks would confound model quality with platform differences in the panel's code-quality comparison.

**Decision.** Retire the independent Axis-A reference build. Each contest round **designates one platform** that all contestants build on — **round 1: Databricks** (canonical Parquet → Unity Catalog; OSI metrics → Metric Views; agent orchestration free per ADR 0005; the webapp's own tech stack remains the contestant's graded choice). A **reproducible instantiation guide is a contest deliverable** (graded under rubric dimension 7); the best output is curated into that platform's Axis-A demonstration + guide. Platform comparison = re-running the contest on another designated platform. Round 1 is explicitly also spec-validation for the platform-native path (OSI→Metric View); spec defects found mid-round are handled by an amendment rule agreed before the round starts.

**Why.** Every token spent now produces graded signal; the leakage and duplication disappear; fixing one platform per round keeps the model comparison unconfounded while the Axis-A demonstration falls out of the same runs. The trade-off — the platform demo is model-built rather than expert-built — is mitigated by curating the best of N attempts and by rubric dimension 4 scoring idiomatic platform use.
